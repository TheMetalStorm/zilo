# zilo
The kilo editor, ported to Zig!

# Motivation
I've been working as a Software Engineer (part-time, but still) for about a year now. And while I've been enjoying my working environment, I often find myself having to work on problems that don't really engage me. Not necesserily because I think I am too "smart" for them, but because I personally do not care about filling a database and then loading data from said database for the 20th time. But maybe that is just me. I love it when the code that I add brings me closer to an actual goal and is not just "more of the same".

Of course there are also very interesting tasks that I get to work on, but even then the feeling I get when I accomplish them is different from the feeling of doing something because I really wanted to do it.

So I am kind of trying to reignite my passion for programming. And what better way to do that than trying to learn a new Language! Originally I was waiting for the release of another language that I've been following for a while. It is called Jai. And while I still plan to experiment with that language once it (eventually, but take your time Jon) releases, I wanted to learn something engaging and new now. That is how I encountered Zig.

I would've become really bored really fast if I had just started solving Leet Code problems or had watched a Youtube Tutorial explaining the syntax of Zig to me. 
But then I remembered a series of youtube videos I had watched a few years ago, in which the Creator tried to port the Text Editor kilo, which is based on nano, in Rust.

For those interested, here is the link to his channel: https://www.youtube.com/@UncleScientist

He was following the Tutorial on https://viewsourcecode.org/snaptoken/kilo/index.html 

I thought to myself: What a great way to learn a language. 
So of course, when the time came for me to think of a project I could work on while learning the basics of Zig, I thought back to that series and my goal became clear: Port to kilo text editor to Zig. 

I am also following the Tutorial on that website, and while Zig is really good at interacting with c and I could mostly straight up copy the code on that website, make a few small changes and call it a port, that would be no fun. I am aiming to use as few calls to c functions as possible (Note that I did not say zero because sometimes I get lazy and just want something to work!)

And thats why I am doing the thing that I am doing! Let's see how far I come!
